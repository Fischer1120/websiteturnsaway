# 05 设计审核与人工复核流程

本文件定义设计审核 agent 如何检查三个前端方案，以及何时交给用户人工复核。

## 审核角色

必须开启独立设计审核子 agent。审核 agent 不应是生成方案的同一个 agent。

审核 agent 的任务：

- 读取 `design/prompt.md`。
- 打开三个设计方案。
- 检查功能覆盖。
- 检查视觉是否符合提示词。
- 给出通过、退回或条件通过结论。

## 审核输入

审核 agent 必须读取：

- [../design/prompt.md](../design/prompt.md)
- [04-frontend-design-agent.md](04-frontend-design-agent.md)
- 三个方案的 `index.html` 或原型入口。
- 三个方案的 `notes.md`。
- 桌面和移动端截图，如果已有。

## 审核标准

每个方案按 100 分审核，80 分以上且无硬性缺失才可通过。

评分维度：

```text
视觉提示词符合度      25
信息架构完整度        20
文章体验              15
图片体验              15
响应式与可用性        15
工艺质量              10
```

硬性缺失包括：

- 没有首页欢迎词。
- 没有导航栏。
- 没有文章栏目或图片栏目。
- 没有文件夹组织呈现。
- 文章详情没有标题、副标题、二级标题或插图。
- 图片详情没有侧边拍摄时间和地点。
- 图片描述没有展示在图片下方。
- 视觉方向明显偏离 `design/prompt.md`。
- 移动端出现明显重叠、溢出或无法阅读。

只要出现硬性缺失，即使总分超过 80，也必须退回。

## 审核流程

1. 读取 `design/prompt.md`，提炼 5 到 8 条不可妥协的视觉检查点。
2. 分别打开三个方案。
3. 对每个方案进行桌面宽度检查。
4. 对每个方案进行移动宽度检查。
5. 逐项确认功能覆盖。
6. 记录截图或观察结果。
7. 给每个方案出具审核报告。
8. 不通过的方案退回前端设计 agent，附带具体修改要求。
9. 退回方案重做后重新审核。
10. 三个方案全部通过后，交给用户人工复核。

## 审核报告模板

```markdown
# Design Review: Concept Name

## Verdict

- Result: Pass / Rework
- Score: 00 / 100

## Prompt Alignment

- 通过点:
- 偏离点:

## Requirement Coverage

- 首页欢迎词:
- 导航栏:
- 文章栏目:
- 图片栏目:
- 文件夹组织:
- 文章详情结构:
- 文章插图:
- 图片侧边元数据:
- 图片下方描述:
- 响应式:

## Required Fixes

1. 问题:
   - 影响:
   - 修改要求:

## Reviewer Notes

- 可保留的设计亮点:
- 需要用户人工判断的审美选择:
```

## 退回规则

退回时，审核 agent 必须提供可执行修改要求，例如：

- 将图片详情页的拍摄时间和地点从底部移到右侧元数据栏。
- 为文章详情补充至少两个二级标题。
- 首页增加明确的文章文件夹和图片文件夹入口。
- 减少通用蓝紫渐变，回到 navy、coral、pastel sunset 和 vintage grain。
- 避免普通博客卡片布局，强化 world exposition poster 和 monumental geometry。

不要只写“风格不符合”。

## 人工复核交付

当三个方案全部通过审核后，总协调 agent 向用户展示：

- 三个方案名称。
- 每个方案的预览链接或截图。
- 每个方案的核心差异。
- 审核结果摘要。
- 推荐选择，但不得代替用户确认。

人工复核前不得进入最终实现。用户确认后，才可进入 [06-build-deploy-sequence.md](06-build-deploy-sequence.md)。
