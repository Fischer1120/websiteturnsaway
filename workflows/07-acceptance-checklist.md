# 07 最终验收清单

本文件用于项目实现后的最终验收。所有项目完成声明都必须以本清单为依据。

## 文档验收

- [ ] 根目录存在 `index.md`。
- [ ] `index.md` 链接到所有流程文档。
- [ ] `workflows/` 下每个流程文档都能独立阅读。
- [ ] 文档明确 Cloudflare、R2、前端设计、审核和部署流程。
- [ ] 文档说明了设计方案必须等待用户人工复核。

## 后端验收

- [ ] 使用 Cloudflare Pages Functions 或 Worker。
- [ ] R2 bucket binding 已配置。
- [ ] 图片原图可以写入 R2。
- [ ] 图片元数据可以写入 R2。
- [ ] API 返回统一成功和错误格式。
- [ ] `/api/admin/*` 路由有鉴权预留。
- [ ] 文章上传接口有路由和 schema 预留。
- [ ] 图片上传接口有路由和 schema 预留。
- [ ] R2 key 不由客户端任意指定。

## 内容模型验收

- [ ] 文章支持 folder。
- [ ] 图片支持 folder。
- [ ] 文章支持 title。
- [ ] 文章支持 subtitle。
- [ ] 文章支持二级标题。
- [ ] 文章支持正文插图。
- [ ] 图片 metadata 包含 capturedAt。
- [ ] 图片 metadata 包含 location。
- [ ] 图片 metadata 包含 description。
- [ ] 图片位置展示有隐私降精度策略。

## 前端验收

- [ ] 首页有欢迎词。
- [ ] 首页有导航栏。
- [ ] 首页有文章栏目。
- [ ] 首页有图片栏目。
- [ ] 文章列表展示文件夹。
- [ ] 图片列表展示文件夹或相册。
- [ ] 文章详情可阅读，并显示标题、副标题和二级标题。
- [ ] 文章详情中的图片正常显示。
- [ ] 图片详情显示大图。
- [ ] 图片详情侧边显示拍摄时间和地点。
- [ ] 图片详情下方显示文本描述。
- [ ] 移动端没有重叠、遮挡或文字溢出。

## 设计流程验收

- [ ] 前端设计使用了 `web-design-engineer` 子 agent。
- [ ] 设计 agent 读取了 `design/prompt.md`。
- [ ] 产出了三个设计方案。
- [ ] 每个方案都有文章案例。
- [ ] 每个方案都有图片案例。
- [ ] 独立审核 agent 检查了三个方案。
- [ ] 不合格方案已退回重做。
- [ ] 三个方案通过审核后才交给用户人工复核。
- [ ] 用户确认后才进入最终实现。

## 视觉验收

- [ ] 能识别出复古未来主义几何建筑方向。
- [ ] 使用了 navy、coral、pastel sunset 或与提示词一致的色彩关系。
- [ ] 巨型几何体、长阴影、world exposition poster 气质有明确呈现。
- [ ] 没有退化为普通博客模板。
- [ ] 没有使用与项目气质不符的通用蓝紫科技渐变。
- [ ] 信息密度、字体尺度和间距在桌面与移动端都稳定。

## 运行验收

- [ ] 本地开发命令可运行。
- [ ] 构建命令可通过。
- [ ] 类型检查可通过。
- [ ] 核心 API 可用。
- [ ] Cloudflare 部署可访问。
- [ ] R2 图片可访问。
- [ ] 404 和错误状态有合理页面或响应。

## 交付说明

最终交付给用户时，应包含：

- 本地预览地址。
- 线上地址，如已部署。
- 选定设计方案名称。
- 已实现功能摘要。
- 未实现但已预留功能。
- 测试和构建结果。
- 需要用户后续提供的内容或账号信息。
